4.指令系统

4.1指令分类

指令：指令是计算机执行某种操作的命令，是计算机运行的最小功能单位。

*计算机只能执行自己系统中的指令，无法执行其它系统的指令。

​ 指令通常包括操作码和地址码两部分，通常有如下分类：

  • 零地址指令

    1. 不需要操作数
    2. 堆栈计算机，例如后缀表达式计算中操作数隐含放在栈顶和次栈顶。

  • 一地址指令

    1. 只需要单操作数，如加1减1，取反，求补等
    2. 需要两个操作数，但其中一个操作数隐含在某个寄存器中。例如乘法操作
      (ACC)OP（A₁）→ACC

  • 二地址指令

    1. 需要两个操作数

  • 三地址指令

  • 四地址指令

指令分类：

  1. 变长指令字结构
  2. 定长指令字结构

按操作码长度分类：

  1. 定长操作码
  2. 可变长操作码

按操作类型分类：

  1. 数据传送
  2. 算数逻辑操作
  3. 移位操作
  4. 转移操作
    • 无条件转移
    • 条件转移
    • 调用和返回
    • 陷阱指令
  5. 输入输出操作

4.2扩展操作码

定长指令字结构+可变长操作码

4.3指令寻址

指令始终由程序计数器PC给出

  • 顺序寻址

    变长指令：程序计数器识别指令前缀判断指令类型后，PC+“1”（根据指令类型，加一个指令长度）

  • 跳跃寻址

    利用在指令之间穿插转移指令（类似goto），指明下一条指令执行的位置

    4.4数据寻址

确定本条指令的地址码指明的真实地址

变指寻址

4.5CISC和RISC

CISC：Complex Instruction Set Computer

设计思路：一条指令完成一个复杂的基本功能

代表：X86架构，主要用于笔记本、台式机等。

RISC：Reduced Instruction Set Computer

设计思路：一个指令完成一个基本“动作“，多条指令组合完成一个复杂的基本功能

代表：ARM架构，主要用于手机、平板。
