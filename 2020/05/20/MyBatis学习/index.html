MyBatis入门

Posted on 2020-05-20

1.MyBatis初认识

  • 把sql语句放在.xml文件中，同java代码分离解耦，修改sql语句不再需要重新修改java代码编译、打包、发布等一系列繁琐过程。
  • 把sql语句集中起来易于管理

2.安装与使用流程

2.1 单独使用

  • 导包：使用maven或者添加jar包到库文件

  • 新建mybatis-config.xml主配置文件

    注意：在xml中很多字符需要转义替换，如&

  • 配置sql语句映射文件

  • 使用

2.2 在spring boot中使用

  • maven坐标（同spring boot使用）

  • 使用流程

    • 需要数据库服务的dao类全部写成接口

    • 对每一个dao类都写一个对应的_mapper.xml，dao类使用的sql语句全部编写在mapper.xml中

    • 指定mapper.xml位置，创建sqlsessionFactory加入到spring中交由mybatis使用，mybatis自动根据mapper.xml和接口生成对应的实现类加入到spring中

    • 配置数据池和包扫描

    • 需要用到dao层时，使用spring中自动生成的实现类

3.mapper.xml的编写

3.1 mapper.xml构成

  • 名称空间namespace

    • 使用接口式自动交由mybatis调用sql语句时，需指定为当前mapper所属的dao层接口所在位置
    • 单独调用时随意填写

  • id:调用时的唯一表示，通常和方法名保持一致。

  • ResultType 指定返回类型的全类名，mybatis会根据数据库表中字段名和类属性名自动封装

  • ParameterType指定方法参数的类型，当多个参数时，此参数自动为map，并把方法的多个参数封装到map中。sql语句中通过#{param1},#{param2}…调用

  • #{id} or ${id}：取出传入的参数

    其中#{ }会进行两次预编译

    [1]第一次只把替换${ }的值，#{ }替换为sql占位符

    [2]第二次再把sql语句的占位符替换

    有效防止sql注入

    • ${ } 通常用作原生sql语句中不支持预编译的部分，如表名
    • #{ } 用作sql语句中占位符部分，底层也是使用PreStatement

4.简单查询规则

4.1 多条记录查返回List

返回值类型照常，多条记录会根据方法返回类型自动封装为List

4.2 多条记录查询返回 map

在Map中会封装结果为value，但较于List还需要一个key，key通常指定为结果中的主键

使用@Mapkey，若没有配置Mapkey，mybatis会认为只返回一个对象。

4.3 联合查询的结果封装

简单情况下查询返回的对象全是基础属性，这是指定resultType即可

但当查询要返回的对象属性不全是基础属性时，意味着有两个实体，其中一个包含另一个，这时候就无法简单指定resultType让mybatis自动封装从数据库查询的结果集。由于mybatis返回的是有map封装的，map里面的属性包括了两个实体的属性，此时需要配置resultMap的映射关系，帮助mybatis完成对象的封装

resultType和resultMap不能同时使用

  • 使用类似book.name的级联属性来映射

  • 使用association标签来关联属性对象（替代上述两行）

  • 使用association标签配置子查询（分步查询）

association使用分步查询时可以配置懒加载，当需要用到这个属性对象才会进行第二步的查询。

这里其实用到了动态代理，封装返回的对象是一个被监控的“假对象”，当我们访问里边的子对象会有前置的处理去查询数据库复制给子对象。

4.4 集合的结果封装

同上述的联合查询，对于对象属性无法直接使用resultType，集合当然不是基础属性，但处理集合有单独的一套方法

  • 在resultMap中配置collection标签
  • 使用分布查询的collection标签

分布查询中需要多个传递多个参数是，使用column=”{bookId=id,booktag=tag}”,同时可使用fetchType=”lazy”属性设置是否懒加载
